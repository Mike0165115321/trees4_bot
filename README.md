## 🌟 ฟีเจอร์เด่นในเวอร์ชัน v6 (Command Center Edition)

- **🇹🇭 ระบบภาษาไทยสมบูรณ์แบบ**: หน้าจอศูนย์ควบคุมและข้อความแจ้งเตือนทั้งหมดเป็นภาษาไทย ใช้งานง่ายสำหรับทุกคน
- **📊 แผงควบคุม (Dashboard) อัจฉริยะ**: ติดตามสถานะรายชื่อเกษตรกร (Pending, Completed, Error) ได้แบบเรียลไทม์
- **⏱️ ระบบวัดขุมพลัง (Speed Tracking Console)**: แผงหน้าปัด (Dashboard) ใหม่ล่าสุดที่ประเมินคะแนนโอกาสการผลิต (ต้น/ชั่วโมง) และความเร็วทำงานของแต่ละบัญชีแบบเรียลไทม์
- **🎯 ฐานข้อมูลเสถียร (SQLite Integration)**: บอกลาไฟล์ CSV และ TXT! ข้อมูลทุกอย่างถูกเก็บไว้ในฐานข้อมูลเดียว ปลอดภัยและเสถียร 100%
- **🚀 ปุ่มบัญชาการ (Start Bot Interface)**: สั่งเริ่มงานหรือหยุดบอทได้ทันทีผ่านหน้าเว็บ (UI-Driven Automation)
- **⚙️ ตัวจัดการการตั้งค่า (Global Settings)**: ปรับสัดส่วนสุ่มคะแนนสุขภาพ (80/15/5) และเปิด-ปิดโหมดเบื้องหลัง (Headless) ได้เพียงปลายนิ้ว
- **💅 ดีไซน์พรีเมียม (Minimalist Analytics UI)**: เลย์เอาต์ Sidebar ทันสมัย พร้อมฟอนต์ Kanit สวยสะอาดตา ลดความซับซ้อนให้เหลือแต่สถิติความเร็วที่สำคัญ

## 🛠️ โครงสร้างระบบ (Architecture)

- **`app.py`**: หัวใจหลักเบื้องหลัง (FastAPI) ที่คอยจัดการ API และหน้าเว็บ
- **`trees_bot.py`**: บอท Playwright สุดอัจฉริยะที่รันตามคิวงานในฐานข้อมูล
- **`database.py`**: ตัวจัดการข้อมูล SQLite (Tables: `accounts`, `settings`)
- **`static/`**: แหล่งรวมขุมพลังดีไซน์ (HTML, CSS, JS)
- **`trees_bot.db`**: ไฟล์ฐานข้อมูลหลักที่เก็บความลับและงานทั้งหมดของคุณ

## ⚙️ อัลกอริทึมการทำงานของบอท (Automation Algorithm Flowchart)

เพื่อความชัดเจน นี่คือกระบวนการคิดและตรรกะเบื้องหลังการทำงาน (Logic) ของบอทผ่านแผนภาพ Flowchart:

```mermaid
graph TD
    A["ผู้ใช้สั่งเริ่มบอท & ตั้งค่า"] -->|Trigger| B{"ค้นหาคิว 'pending' จาก SQLite"}
    B -- ไม่พบคิว --> C["หยุดพักและรอรอบคำสั่งใหม่"]
    B -- พบคิวงาน --> D(("Playwright Launch Browser"))
    
    D --> E["Login เข้าสู่ระบบ Trees4All"]
    E --> F["คลิกเจาะผ่าน Vuetify Layer & Popup"]
    F --> G["กรอกข้อมูลฟอร์ม และ ปักหมุด Geo-Map"]
    
    G --> H["ดึงไฟล์รูปจาก Local มาผ่าน DataTransfer"]
    H --> I["แนบไฟล์ (I/O Upload) และกดเซฟ"]
    
    I --> J{"ระบบเว็บตอบสนองสำเร็จหรือไม่?"}
    J -- บันทึกเว็บสำเร็จ --> K["Update DB -> 'done'"]
    J -- ขัดข้อง/Timeout --> L["Update DB -> 'error'"]
    
    K --> M["หน่วงเวลาแบบมนุษย์ (Human Delay)"]
    L --> M
    M --> B
```

## 🚀 วิธีเริ่มต้นใช้งานใน 3 ขั้นตอน

1.  **รันระบบ**:
    ```bash
    python app.py
    ```
2.  **เปิด Command Center**: เข้าสู่เบราว์เซอร์แล้วพิมพ์ `http://localhost:8000`
3.  **จัดการงาน**:
    - เพิ่มบัญชีเกษตรกรง่ายๆ ในเมนู **"เพิ่มบัญชีใหม่"**
    - ตั้งค่าสุขภาพที่ต้องการในเมนู **"ตั้งค่าระบบ"**
    - กดปุ่ม **"เริ่มรันบอท 🚀"** ในหน้าแผงควบคุม แล้วปล่อยให้ AI จัดการงานแทนคุณ!

## 💡 คำแนะนำเพิ่มเติม (Tips)
- **โหมดเบื้องหลัง (Headless)**: หากต้องการให้บอทรันเงียบๆ ไม่กวนสายตา ให้เปิดใช้ในหน้า Settings
- **การกู้คืนคิว**: หากอินเทอร์เน็ตหลุด บอทจะไม่เริ่มใหม่ตั้งแต่ต้น แต่จะข้ามคนที่เป็น 'สำเร็จแล้ว' ไปทำคิวที่เหลือให้ทันทีครับ

## ⚖️ สถิติและประสิทธิภาพ (Human vs. Bot Performance)

เพื่อให้เห็นภาพที่ชัดเจนถึงกระบวนการทำงานของ Algorithm และขุมพลัง Automation ที่เปลี่ยนกระบวนการที่ยาวนานให้จบในอึดใจ:

| เกณฑ์การเปรียบเทียบ | ทรัพยากรมนุษย์ (Manual Input) | Trees4All Bot 🤖 (Automated) |
|:---|:---|:---|
| **ความเร็วในการบันทึก** | 2 - 3 ต้น / นาที | **6.69 ต้น / นาที** (ทดสอบจากสภาวะการใช้งานจริง) |
| **ผลผลิตสะสมรายชั่วโมง** | 60 - 80 ต้น | **~400 ต้น** (เร็วกว่าและเสถียรกว่าหลายอึดใจ) |
| **ความแม่นยำของข้อมูล** | อาศัยความจำ มีโอกาสผิดพลาดสูง | **แม่นยำ 100%** คำนวณและประมวลผลจากตัวเลขเป๊ะๆ ไร้ข้อกังขา |
| **การจัดการภาพถ่าย** | ต้องเลือกไฟล์ทีละครั้ง มีโอกาสสับสน | **ระบบ UUID กึ่งออโต้** ควบคุมรูปรวดเดียวจบไม่มีหลง |
| **ขีดจำกัดการประมวลผล** | 3 - 4 ชั่วโมง (เหนื่อยล้า/ความแม่นยำตก) | **24 ชั่วโมง / 7 วัน** (ทำงานได้เรื่อยๆ ไม่มีวันพัก ไม่ขอบ่น) |

> **Business Value:** ระบบนี้ถูกออกแบบมาเพื่อ **ซื้อเวลาทำงาน** ให้กลับคืนมา ลดโอกาสเกิดข้อผิดพลาดจากมนุษย์ (Human Error) เป็นศูนย์ และเพิ่มผลผลิตระดับองค์กรได้อย่างมีนัยสำคัญ
