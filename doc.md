# [แผนงาน] ระบบอัปโหลดรูปภาพอัตโนมัติ (Automated Image Upload System)

แผนงานนี้จัดทำขึ้นเพื่อเตรียมความพร้อมสำหรับการเพิ่มฟีเจอร์อัปโหลดรูปภาพต้นไม้ในเวอร์ชันถัดไป โดยเน้นการบริหารจัดการผ่าน Web UI และการทำงานอัตโนมัติของบอทแบบทีละภาพจนครบงาน

---

## 🏗️ สรุปภาพรวมระบบปัจจุบัน (Current System Overview)
ปัจจุบันระบบ **Trees4All Command Center v6** ทำงานบนพื้นฐานดังนี้:
1. **Frontend (Vanilla HTML/JS)**: หน้าเว็บ Command Center สำหรับจัดการคิวรายชื่อเกษตรกรและตั้งค่า Global Settings แบบเรียลไทม์
2. **Backend (FastAPI)**: ทำหน้าที่เป็นตัวกลางรับส่งข้อมูลระหว่างหน้าเว็บ ฐานข้อมูล และสั่งรันบอท
3. **Database (SQLite)**: เก็บข้อมูลทุกอย่างที่ `trees_bot.db` โดยมีตาราง `accounts` (เก็บเบอร์โทร, รหัส, ผู้สำรวจ) และ `settings` (เก็บค่าสุขภาพ, โหมดการรัน)
4. **Automation Core (Playwright)**: บอทจะดึงคิวที่สถานะ `pending` จาก DB มา Login -> กรอกข้อมูลพิกัดต้นไม้ทีละต้น จนครบหลักฐาน -> อัปเดตสถานะเป็น `done`

---

## 📋 แผนการดำเนินงานในอนาคต (Future Phase: Image Integration)

### 1. [DATABASE] ปรับปรุงโครงสร้างฐานข้อมูล
เพื่อรองรับการผูกรูปภาพหลายใบเข้ากับบัญชีเดียว เราต้องเพิ่มตารางใหม่:
#### [NEW] Table `images`
- `id`: Primary Key
- `account_id`: Foreign Key เชื่อมกับตาราง `accounts`
- `file_path`: พาธที่เก็บไฟล์รูปภาพในเครื่อง
- `status`: สถานะการอัปโหลด (`pending`, `done`)

---

### 2. [WEB UI] ส่วนจัดการรูปภาพ
ต้องเพิ่มฟีเจอร์ในหน้า **เพิ่มบัญชีใหม่** (หรือปุ่มจัดการรูปในตารางคิว):
- **Image Uploader**: ช่องเลือกรูปภาพพร้อมกันได้หลายใบ (Multiple Selection)
- **Preview System**: แสดงรูปตัวอย่างก่อนบันทึก
- **API (FastAPI)**: สร้าง Endpoint `/api/upload` เพื่อรับไฟล์และบันทึกลงโฟลเดอร์ `uploads/` พร้อมบันทึกพาธลง Database

---

### 3. [BOT LOGIC] ปรับปรุงขั้นตอนการทำงานของบอท
แก้ไขไฟล์ `trees_bot.py` เพื่อแทรกขั้นตอนนี้ในจุดสุดท้ายของแต่ละคน:

#### ขั้นตอนที่เพิ่มเข้ามา (Image Upload Loop):
1. **Detection**: หลังจากกรอกรหัสต้นไม้ต้นสุดท้ายเสร็จ บอทจะเช็คใน DB ว่าบัญชีนี้มีรูปภาพที่ต้องอัปโหลดกี่รูป
2. **Navigation**: บอทจะรอจนกว่าจะถึงหน้าอัปโหลดรูป (จุดที่ไม่มีช่องให้กรอกข้อมูลแล้ว)
3. **Iteration**: วนลูปอัปโหลดรูปภาพทีละใบ:
   - คลิกปุ่มเลือกรูปภาพ
   - ใส่พาธไฟล์ (`file_path`) จาก Database
   - รอจนกว่าระบบเว็บจะอัปโหลดใบนั้นเสร็จ
   - ทำซ้ำจนครบทุกใบในคิว
4. **Finish**: เมื่อครบแล้วจึงค่อยคลิกปุ่ม "เสร็จสิ้น" (Finish) เพื่อจบงานของคนนั้นจริง ๆ

---

## 🛠️ รายละเอียดทางเทคนิคที่ต้องเตรียม
> [!IMPORTANT]
> **การจัดการโฟลเดอร์:** ต้องสร้างโฟลเดอร์ `static/uploads/` เพื่อเก็บไฟล์รูปภาพ และตรวจสอบว่าชื่อไฟล์ไม่ซ้ำกัน (เช่น ใช้ Timestamp นำหน้าชื่อไฟล์)

> [!TIP]
> **Bot Performance:** ในขั้นตอนการอัปโหลดรูป บอทควรเพิ่ม `asyncio.sleep` หรือรอ `wait_for_selector` ที่บอกว่าอัปโหลดเสร็จแล้ว (เช่น แถบ Progress หายไป) เพื่อป้องกันบอทกดยืนยันก่อนที่รูปจะเข้าสู่ระบบจริง

1. **จำนวนรูปสูงสุด:** จำกัดจำนวนรูปภาพต่อ 1 บัญชีหรือไม่? (เช่น ไม่เกิน 10 รูป)
2. **ขนาดไฟล์:** ต้องมีการบีบอัดรูปภาพ (Compression) ก่อนอัปโหลดเพื่อประหยัดพื้นที่เซิร์ฟเวอร์หรือไม่?
3. **Trigger Point:** หน้าอัปโหลดรูปภาพในเว็บ Trees4All แสดงผลทันทีที่จบการกรอกพิกัด ใช่หรือไม่?
