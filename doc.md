# [คู่มือ] ระบบ Trees4All Command Center v6 (เสร็จสมบูรณ์)

เอกสารฉบับนี้สรุปภาพรวมของระบบหลังจากได้รับการพัฒนาจนครบถ้วนตามความต้องการ (Full Feature Set) พร้อมใช้งานสำหรับการทำงานระดับ Enterprise

---

## 🏗️ ภาพรวมระบบปัจจุบัน (Complete System Overview)
ปัจจุบันระบบ **Trees4All Command Center v6** ทำงานบนมาตรฐานสูงสุดดังนี้:

### 1. Frontend (ความงามและการควบคุม)
- **Modern Dashboard**: แสดงสถิติความเร็วแบบ Real-time และตารางจัดการคิวแบบแยกส่วน (Pending/Finished)
- **Live System Logs**: หน้าต่างแสดง Log การทำงานแบบสี (Color-coded) พร้อมไฟสถานะ Live ทำให้ไม่ต้องเปิด Terminal ตรวจสอบ
- **Smart Controls**: ปุ่มควบคุม "พักบอท ⏸️", "ทำต่อ ▶️", และ "หยุดบอท 🛑" ที่ทำงานแบบสัมพันธ์กับตัวบอทจริง

### 2. Backend (ศูนย์กลางการประมวลผล)
- **FastAPI Core**: จัดการคิวงาน, การตั้งค่า Global Settings, และระบบ Logging
- **Graceful Shutdown**: ระบบประมวลผลความละเอียดสูงที่สามารถสั่ง "หยุดบอท" ได้อย่างปลอดภัย (จบคิวที่ทำค้างอยู่ก่อนปิด) หรือบังคับปิด (Force) หากจำเป็น

### 3. Database (ความปลอดภัยของข้อมูล)
- **SQLite Engine**: เก็บข้อมูลบัญชี, การตั้งค่า, และประวัติความเร็วรายตัว
- **Retry Logic**: เพิ่มระบบ "รันเฉพาะที่พลาด (Retry Error)" เพื่อดึงเฉพาะคิวที่ล้มเหลวกลับมาทำงานใหม่ โดยไม่รบกวนบัญชีที่สำเร็จแล้ว

### 4. Automation Core & Image Upload (กลไกหลัก)
- **Playwright Engine**: ทำหน้าที่ล็อกอิน, ข้ามป๊อปอัป, และกรอกพิกัดพรรณไม้แบบอัตโนมัติ 100%
- **Sequential Image Upload**: ระบบอัปโหลดรูปภาพหลักฐานทีละใบหลังจากกรอกพิกัดเสร็จ โดยจะส่งรูปภาพและกดยืนยันทีละขั้นตอนจนครบถ้วนตามฐานข้อมูล

---

## 🛠️ ขั้นตอนการทำงานของบอท (Final Bot Workflow)
1. **Login Phase**: เข้าสู่ระบบและปูพื้นฐานพิกัดแปลง
2. **Recording Phase**: วนลูปกรอกพิกัดต้นไม้ต้นที่ 1 ถึง N จนครบ
3. **Closing Phase**: กด "เสร็จสิ้น" ในหน้าพิกัดเพื่อบันทึกข้อมูลหลัก
4. **Image Phase [NEW]**: 
   - ดึงรายการรูปภาพจาก DB
   - คลิกเข้าเมนู "ถ่ายรูปแปลงต้นไม้"
   - วนลูป อัปโหลด -> กดเสร็จสิ้น ทีละภาพจนหมดคิว
5. **Completion**: อัปเดตสถานะเป็น `done` และผ่านไปยังบัญชีรายชื่อถัดไป

---

## 🛡️ การดูแลรักษา (Maintenance)
- **Logs**: หากบอทติดขัด สามารถดูสาเหตุได้ทันทีจากหน้าจอ Dashboard
- **Pause**: หากต้องการแทรกแซงหรือปรับเปลี่ยนข้อมูลหน้าเว็บสด สามารถกดปุ่ม "พักบอท" ได้ทุุกเมื่อโดยไม่เสีย Session
- **Images**: รูปภาพที่ถูกอัปโหลดจะถูกเก็บไว้ที่ `static/uploads/` และสามารถจัดการผ่าน Web UI ได้โดยตรง
