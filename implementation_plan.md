# แผนการสร้างแอปพลิเคชันสำหรับ Windows (.exe)

โปรเจคปัจจุบันเป็นเว็บแอปพลิเคชัน (FastAPI + HTML) ที่รันบอท (Playwright) แผนนี้จะอธิบายขั้นตอนการรวบรวมส่วนประกอบทั้งหมดเข้าด้วยกันเพื่อให้ใครก็สามารถนำไปใช้งานได้เพียงแค่เปิดไฟล์เดียว

## Proposed Changes

การเปลี่ยนแปลงจะแบ่งเป็น 3 ระยะหลัก: เตรียมโค้ด, สร้างตัวเชื่อมต่อ (Wrapper), และช่วงการแพ็คไฟล์ (Building)

---

### Phase 1: การเตรียมโค้ดเพื่อรองรับการเป็น EXE

เพื่อให้แอปหาหน้าเว็บและฐานข้อมูลเจอเมื่ออยู่ในไฟล์เดี่ยว เราต้องแก้การเรียก Path ในโค้ด

#### [MODIFY] [database.py](file:///c:/Users/Gigabyte/Downloads/files/trees_bot_v6/database.py)
*   ปรับให้ฐานข้อมูล `trees_bot.db` ถูกเก็บไว้ในโฟลเดอร์เดียวกับไฟล์ EXE เสมอ (เพื่อไม่ให้ข้อมูลหายทุกครั้งที่เปิดแอป)

#### [MODIFY] [app.py](file:///c:/Users/Gigabyte/Downloads/files/trees_bot_v6/app.py)
*   ปรับเปลี่ยนการเรียกโฟลเดอร์ `static` ให้รองรับระบบ `sys._MEIPASS` ของ PyInstaller (ระบบดึงไฟล์จาก temp ตอนรันแอป)

---

### Phase 2: การสร้างหน้าต่างโปรแกรม (The App Wrapper)

สร้างหน้าต่างโปรแกรมเพื่อให้ดูเป็นแอปจริงๆ ไม่ต้องเปิดผ่าน Browser เอง

#### [NEW] [main.py](file:///c:/Users/Gigabyte/Downloads/files/trees_bot_v6/main.py)
*   **Thread 1 (Backend):** รัน FastAPI Server ในพื้นหลัง
*   **Thread 2 (UI):** ใช้ `pywebview` เปิดหน้าต่างโปรแกรมขึ้นมา และโหลดหน้าเว็บจาก `localhost:8000`

---

### Phase 3: การรวบรวมไฟล์ (Packaging with PyInstaller)

ขั้นตอนนี้คือการนำทุกอย่างมารวมกัน โดยเราจะเลือกแบบ **Self-contained (รวม Browser เข้าไปด้วย)**

1.  **รวม Playwright Browser:** ติดตั้ง Chromium ไว้ในโฟลเดอร์โปรเจคและสั่งให้ PyInstaller รวมมันเข้าไปด้วย
2.  **Config ข้อมูล:** กำหนดให้รวมโฟลเดอร์ `static/` และไฟล์ Python อื่นๆ
3.  **สร้าง EXE:** รันคำสั่ง Build เพื่อให้ได้ไฟล์เดี่ยวออกมา

---

## Verification Plan

### Automated Verification
*   ทดสอบรัน `main.py` ในเครื่องก่อนแพ็ค เพื่อดูว่าหน้าต่างแอปเปิดขึ้นมาปกติและบอททำงานได้ไหม
*   ใช้คำสั่ง `pyinstaller` ตรวจสอบว่าไม่มี Error ระหว่างการ Build

### Manual Verification
*   **ทดสอบการใช้งาน:** ดับเบิลคลิกไฟล์ `.exe` ที่ได้
*   **ตรวจสอบความถูกต้อง:** ลองเพิ่มบัญชี, ตั้งค่า และกดเริ่มบอท เพื่อดูว่า Playwright รันขึ้นมาได้ถูกต้องจากภายในตัว EXE หรือไม่

## Open Questions
> [!IMPORTANT]
> 1. **ไอคอนของแอป:** คุณมีไฟล์รูปภาพ (.ico) ที่ต้องการใช้เป็นไอคอนของแอปไหมครับ? ถ้าไม่มีผมจะใช้ไอคอนมาตรฐานไปก่อน
> 2. **ความต้องการของ Windows:** ผู้ใช้จำเป็นต้องมี Permission ในการเขียนไฟล์ในโฟลเดอร์ที่วางแอป (เพื่อสร้างไฟล์ database) ควรแนะนำให้วางแอปไว้ที่ Desktop หรือโฟลเดอร์ส่วนตัว ไม่ควรใส่ไว้ใน C:\Program Files โดยตรงหากไม่มีสิทธิ์ Admin
